Plattform

Jag har valt att använda Render ihop med Github för att automatisera min publiceringsprocess utan FTP.

Om Render

Render är en molntjänst som går att använda för att hosta flera olika typer av webbapplikationer.

  • Render marknadsför sig bland annat som "det snabbaste sättet att hosta alla sina webbapplikationer"

  • Render har stöd för flera programmeringsspråk och verktyg, bland annat JavaScript, Node, Docker, Python, Ruby, Go, Rust och PHP

  • Render har olika betalplaner med första steget som gratis

  • Jag valde Render för att de jämfört med många andra alternativ erbjuder många flera finesser även under gratis-varianten

  • Render har bra dokumentation

  • Render gör det busenkelt att registrera sig bland annat med stöd via Github-konto

Publiceringsprocess

Render gör det väldigt enkelt att publicera sina webbapplikationer direkt från välkomstskärmen efter registrering.

  • Från välkomstskärmen kan man välja att publicera en statisk webbplats

  • När man valt statisk webbplats blir man presenterad med en vy där man kan välja Github-repo

  • När man valt statisk webbplats blir man presenterad med en vy där man kan välja Github-repo

  • Man kan även söka efter specifikt Github-repo med hjälp av URL

  • När man väljer ett Github-repo får man ange lite inställningar, så som "root directory", eventuellt "build"-kommando om man vill köra Webpack-jobb eller dylikt på servern

  • Publiceringsprocessen tog mig 5-10 minuter maximalt efter registrering, med andra ord en väldigt användarvänlig plattform
